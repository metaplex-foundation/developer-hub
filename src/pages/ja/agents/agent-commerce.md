Metaplex上のエージェントコマースは、登録されたエージェントの生産的な経済活動です — EIP-8004メタデータを介して互いを発見し、サービスに料金を請求し、ステーブルコインで相手に支払い、エグゼクティブが署名したトランザクションを通じてオンチェーンで決済します。[エージェントファイナンス](/agents/agent-finance)がエージェントの資本化方法をカバーするのに対し、エージェントコマースはエージェントがその資本で何をするか、そしてどのように生計を立てるかをカバーします。{% .lead %}

## 概要

登録されたMetaplexエージェントは、初日から生産的なコマースのための構成要素を備えています: 検証可能なID、サービスを宣伝するEIP-8004準拠の登録ドキュメント、任意のSPLトークンを保持して使用できるPDAウォレット、そしてアセット所有者がいつでも取り消すことができるアセットごとの実行委任。

- **デフォルトでEIP-8004**: エージェント登録はEIP-8004メタデータ (`type: "https://eips.ethereum.org/EIPS/eip-8004#registration-v1"`) を発行するため、MetaplexエージェントはあらゆるEIP-8004コンシューマーと相互運用可能
- **サービス発見**: すべての登録は、エンドポイント、バージョン、スキル、ドメインを持つ`services[]`配列を宣伝します。相手側は登録URIを取得して機能を発見します
- **x402サポートフラグ**: エージェントメタデータには、HTTP 402ステーブルコイン決済クライアントがエージェントが機械間決済用にセットアップされているかどうかを発見できるように、ファーストクラスの`x402Support`ブール値が含まれます
- **エグゼクティブ委任**: [`mpl-agent-tools`](/smart-contracts/mpl-agent/tools)はアセットごとに`ExecutionDelegateRecordV1` PDAを発行し、エグゼクティブはエージェントに代わって支払いに署名でき、所有者はいつでも取り消すことができます

{% callout type="note" title="エージェントコマース vs. エージェントファイナンス" %}
**エージェントコマース**は*生産的な活動*に関するものです — エージェントがどのように相手側を発見し、収益を得て、支払い、取引するか。**[エージェントファイナンス](/agents/agent-finance)**は*資本化とガバナンス*に関するものです — エージェントがどのように資金調達され、ホルダーがそのミッションとどのように整合するか。ファイナンスはエージェントをブートストラップし、コマースはエージェントが自立する方法です。
{% /callout %}

## Metaplexエージェントコマースのプリミティブ

エージェントコマースのすべてのレイヤーはMetaplexプリミティブとして出荷されます — オンチェーンID、EIP-8004メタデータ、Asset Signerウォレット、エグゼクティブ委任、正規のトークンバインディング:

| プリミティブ | 場所 | 提供する機能 |
|-----------|----------------|-----------------|
| **オンチェーンID** | MPL Coreアセットにバインドされた[`AgentIdentityV2`](/smart-contracts/mpl-agent/identity) PDA | 相手側はエージェントのIDをドメインやウォレットだけでなくオンチェーンで検証 |
| **EIP-8004メタデータ** | `agentMetadataUri`にあるオフチェーンJSON、スキーマは[`agent-metadata.ts`](https://github.com/metaplex-foundation/genesis-app)に | クロスプラットフォームのサービス発見と機能の宣伝 |
| **PDAウォレット (Asset Signer)** | [MPL Core](/smart-contracts/core)によって派生されたシード `["mpl-core-execute", asset]` | 任意のSPLトークンを保持して使用; 秘密鍵なし |
| **エグゼクティブ委任** | `mpl-agent-tools`の[`ExecutionDelegateRecordV1`](/smart-contracts/mpl-agent/tools) PDA | オフチェーンオペレーターがエージェントに代わって署名; アセットごと; 取り消し可能 |
| **トークンバインディング** | `AgentIdentityV2`上の[`setAgentTokenV1`](/dev-tools/cli/agents/set-agent-token) | エージェントとその[トークン](/agents/agent-finance)間の永続的なリンク (収益ルーティング用) |

これらの上に決済プロトコル (x402スタイルのフロー) とより豊富なエージェント間連携をレイヤー化することはランタイム統合です — オンチェーンプリミティブはすでに整っています。

## EIP-8004 はすぐに使える

MetaplexのCLIまたはLaunchpadを介して登録されたすべてのエージェントは、EIP-8004準拠のメタデータドキュメントを発行します。`type`フィールドはデフォルトで:

```
https://eips.ethereum.org/EIPS/eip-8004#registration-v1
```

メタデータスキーマには以下が含まれます:

| フィールド | 目的 |
|-------|---------|
| `name`、`description`、`image` | 人間が読めるID |
| `services[]` | 各サービスには`name`、`endpoint`、`version`、`skills[]`、`domains[]`があります |
| `x402Support` | エージェントがHTTP 402ステーブルコイン決済を受け入れるかどうか |
| `active` | エージェントが現在運用中かどうか |
| `registrations[]` | クロスレジストリ登録 (エントリごとに`agentId` + `agentRegistry`) |
| `supportedTrust[]` | エージェントが宣言する信頼メカニズム (CLIは`"reputation"`、`"crypto-economic"`、`"tee-attestation"`を提供) |

相手側 (人間またはエージェント) は、エージェントの`agentMetadataUri`を取得することでこのすべてを発見します — URIはCoreアセットに添付された`AgentIdentity`プラグインにオンチェーンで記録されます。

### サービスと信頼宣言での登録

Metaplex CLIは、サービスと信頼の登録を直接公開します:

```bash {% title="発見可能なサービスと信頼メカニズムを持つエージェントを登録" %}
mplx agents register --new \
  --name "My Agent" \
  --description "What my agent does" \
  --services '[{"name":"MCP","endpoint":"https://myagent.com/mcp","skills":["analysis","summarization"]}]' \
  --supported-trust '["reputation","tee-attestation"]' \
  --json
```

登録後、誰でもエージェントのオンチェーン`agentMetadataUri`からメタデータを解決し、宣伝されたエンドポイントにリクエストをルーティングできます。

## x402: スタブではなくフラグによるステーブルコイン決済

[x402](https://www.x402.org)は、HTTP `402 Payment Required`を使用してステーブルコインのマイクロペイメントをAPIアクセスのファーストクラス部分にする新興プロトコルです。クライアントはリソースをリクエストし、`402`と支払い指示を受け取り、オンチェーンで決済し、支払い証明と共に再試行します。

Metaplexはx402サーバーやクライアントを出荷しません — それはランタイムの問題です。出荷するのは、プロトコルが*エージェント*側から必要とするすべてです:

- **`x402Support: true`** をメタデータに設定して、呼び出し元がx402機能を発見できるようにします
- **USDC/USDTを保持するPDAウォレット** — Asset Signerは任意のSPLトークンを受け入れます
- **送金に署名できるエグゼクティブ** がCoreのExecuteフックを介して、エージェントが必要とするAPI呼び出しやリソースの料金を支払います

つまり、オンチェーンの信頼と署名のプリミティブはすでに整っており、それらをx402サーバーフレームワークに配線することは統合タスクであり、オンチェーンプロトコル設計タスクではありません。

## サービス発見によるエージェント間連携

エージェント間の領域 (「A2Aプロトコル」というバナーの下でしばしば議論される) は、機能の宣伝、サービス発見、委任された作業に対する支払いという小さなニーズの集合に収束しています。Metaplexの既存のプリミティブは最初の2つに直接マッピングされます:

- **機能の宣伝** — `services[].skills`と`services[].domains`がエージェントが行うことを宣言
- **サービス発見** — `agentMetadataUri`を取得すると、エンドポイント、バージョン、プロトコル情報が返されます。エージェントはMetaplex登録をインデックス化してディレクトリを構築できます
- **委任された作業の支払い** — エージェントのPDAウォレットが相手側エージェントのPDAウォレットに任意のSPLトークンで支払います。両方のトランザクションはそれぞれのエグゼクティブによって署名されます

クロスレジストリの相互運用性は`registrations[]`フィールドを通じてサポートされており、Metaplexエージェントが他のレジストリ (たとえばEVMサイドのERC-8004登録) に並行登録を宣言できるようにし、エコシステム全体で単一の信頼できる情報源を維持します。

## Metaplexエージェントが支払いを決済する方法

完全なコマースフローでは、これらのプリミティブをエンドツーエンドで使用します:

1. **相手側の発見** — クライアント (人間またはエージェント) が対象エージェントの`agentMetadataUri`を取得し、`services[]`、`x402Support`、`supportedTrust[]`を読み取ります
2. **サービスリクエスト** — クライアントが宣伝されたエンドポイントを呼び出します
3. **支払い** — 有料サービスの場合、サーバーはHTTP 402 (または類似のゲーティング) を返します。クライアントはエージェントの[Asset Signer PDA](/agents/what-is-an-agent)にUSDCまたは別のステーブルコインで支払います
4. **検証** — サーバーはオンチェーンの支払いを読み取り、送信者 (オプションで送信者自身のエージェント登録を信頼スコアリング用に) を確認し、リソースのロックを解除します
5. **送金** — エージェント自身が相手 (コンピュート、データ、他のエージェント) に支払う必要がある場合、その[エグゼクティブ](/agents/run-an-agent)はCore Execute命令でラップされた送金に署名します

各ステップはMetaplexスタックがすでに提供しているプリミティブを使用します。オフチェーンのカストディアンやプラットフォーム仲介のエスクローはありません。

## 注意事項

- このページはMetaplexが今日出荷している構成要素を説明します。x402サーバーとインデックス化されたエージェントディレクトリのオンボーディングフローは別のランタイムの問題であり、独自のガイドが用意されます
- EIP-8004はメタデータ形式です。[エージェントファイナンス](/agents/agent-finance)とエージェントコマースはその上のレイヤーです。同じ登録ドキュメントが両方によって読み取られます
- `AgentIdentityV2`の`agentToken`フィールドは[`setAgentTokenV1`](/dev-tools/cli/agents/set-agent-token)を介して一度設定され、永続的です。エージェントのトークンホルダーへの収益ルーティングはファイナンスの問題です。コマースフローはSOLまたはステーブルコインをエージェントのPDAに直接ルーティングできます
- アセット所有者はいつでもエグゼクティブを取り消すことができます。これは自律的な支払い権限を委任する際の安全弁です

## FAQ

Metaplexのエージェントコマースに関する一般的な質問。

### エージェントコマースとは何ですか？
エージェントコマースは、自律型AIエージェントの生産的な経済活動です — オンチェーンで収益を得たり、サービスの料金を支払ったり、他のエージェントや人間と取引したりすること。エージェントが経済参加者としてどのように行動するかをカバーし、エージェントがどのように資金調達されるかについてではありません。

### エージェントコマースはエージェントファイナンスとどう違いますか？
[エージェントファイナンス](/agents/agent-finance)は、エージェントがどのようにトークンを通じて**資本化およびガバナンス**されるかをカバーします。エージェントコマースは、エージェントがその後どのように**収益を得て、支払い、取引するか**をカバーします。ファイナンスはエージェントに資金を提供し、コマースはエージェントがその資金で行うことです。

### MetaplexエージェントはEIP-8004互換ですか？
はい。デフォルトのメタデータ`type`は`https://eips.ethereum.org/EIPS/eip-8004#registration-v1`です。すべてのMetaplexエージェント登録は、`services[]`、`x402Support`、`supportedTrust[]`、`registrations[]`フィールドを持つEIP-8004準拠のドキュメントを発行します。EIP-8004メタデータを消費するものなら何でもMetaplexエージェントを消費できます。

### Metaplexはx402決済をサポートしていますか？
エージェントメタデータには、機能発見のためのファーストクラスの`x402Support`ブール値があり、PDAウォレットはすでに任意のSPLトークン (USDCを含む) を受信でき、エグゼクティブは送金トランザクションに署名できます。プロトコルレイヤー (x402サーバーフレームワーク) は、これらのプリミティブの上に位置するランタイム統合です。

### エージェントはMetaplex上でどのように互いを発見しますか？
すべての登録済みエージェントには、EIP-8004メタデータを含む公開登録URIがあります。相手側エージェントはオンチェーンの`AgentIdentity`プラグインからこのURIを解決し、`services[].endpoint`、`skills`、`domains`、サポートされているプロトコルを読み取って、リクエストの送信先と方法を決定します。

### Metaplexエージェントは今日収益を得ることができますか？
はい。エージェントのPDAウォレット (Asset Signer、`["mpl-core-execute", asset]`として派生) は任意のSPLトークンを受け入れます。秘密鍵はありません — ウォレットはCoreのExecuteライフサイクルフックを通じてのみ制御され、[`mpl-agent-tools`](/smart-contracts/mpl-agent/tools)を介してアセットのエグゼクティブによって署名されます。

### エージェントは自律的にサービスに対してどのように支払いますか？
エグゼクティブはCoreの[Executeライフサイクルフック](/smart-contracts/core/execute-asset-signing)を通じて送金トランザクションに署名します。支払いゲート付きAPIの場合、エージェントの決済クライアント (x402またはその他) は転送を構築し、エグゼクティブが署名し、APIサーバーはリソースのロックを解除する前にオンチェーンの支払いを検証します。
